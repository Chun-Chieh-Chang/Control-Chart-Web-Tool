// turbo-all
這是根據專案全局規範設定的標準操作程序，每次修改代碼後應遵循此流程。

### 步驟 1：代碼精準修訂
1. 根據需求進行修改，遵循 **精準比速度更重要** 的原則。
2. 確保 JavaScript 符合 **ES6+** 規範。
3. 保持 **Glassmorphism** 視覺風格。

### 步驟 2：跑通測試
1. 啟動本地開發伺服器（如果適用，例如 `npm run dev`）。
2. 使用瀏覽器工具或自動化腳本驗證修改。
3. **必須確認功能完全正常運作** 才能進入下一步。

### 步驟 3：總結經驗與記錄
1. 總結本次修訂的核心內容。
2. 記錄遇到的錯誤、解決過程與「經驗教訓」。
3. 確保文件符合 **MECE** 原則。

### 步驟 4：詢問 GitHub 推送
1. 主動詢問使用者：「代碼已通過測試且完成總結，是否需要立即推送到 GitHub？」
2. 若使用者同意，執行 Git Commit 與 Push。
