# 開發紀錄（2026-02-03）

## 問題描述
- 匯入頁面「點擊欲分析項目」卡片中的「目標值」顯示為 B2 (USL)；正確應為 A2 (Target)

## 根因分析
- 匯入預覽卡片的目標值來源使用 SheetJS `sheet_to_json({ header: 1 })` 取得的二維陣列
- 目標值讀取索引誤設為 `data[1][1]`（B2），應為 `data[1][0]`（A2）
- 分析流程與規格提取模組（如 `SpecificationExtractor`）不受影響；僅匯入預覽卡片顯示邏輯出錯

## 修正措施
- 精準修改：將 `js/app.js` 中 `showInspectionItems` 讀取目標值的索引改為 A2
- 變更：
  - 由 `data[1][1]` 改為 `data[1][0]`

## 驗證步驟
- 於瀏覽器開啟專案首頁（`npm run start` 會開啟 `index.html`）
- 切換至「匯入QIP數據」頁面，確認「點擊欲分析項目」卡片的「目標值」來源為 A2
- 以瀏覽器開發者工具檢查 Console：無錯誤

## 影響範圍
- 僅影響匯入頁面之項目卡片的目標值顯示
- 與後續分析（批次、穴號、群組）計算及圖表無副作用
